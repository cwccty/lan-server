# 联机助手

面向普通 PC / Steam 玩家的 Windows 小型游戏联机辅助工具。

> 当前项目仍处于 `0.1.0 早期公开测试版` 阶段。它不是“所有游戏一键联机”的承诺版，而是先把 n2n 组网、端口代理、UDP 广播桥、游戏方案、Terraria 服务端托管和诊断报告整理到一个客户端中，逐步沉淀可复用方案。

## 1. 项目定位

很多小型游戏的联机问题并不是单一问题，而是由以下环节组合造成：

```text
没有公网 IP
NAT / 防火墙阻挡
局域网广播无法跨地域
游戏服务端只监听本机
不同游戏的加入方式不同
玩家不会配置端口、虚拟 IP、服务端和邀请信息
```

联机助手希望把这些步骤整理成一个更容易理解的流程：

```text
游戏扫描
→ 匹配适配器
→ 推荐联机方案
→ 启动 n2n 组网
→ 按需启动 TCP / UDP / UDP 广播桥
→ 生成邀请好友包
→ 用诊断报告定位失败原因
```

## 3. 关于“Terraria 向导”和其他游戏

Terraria 向导不是未来每个游戏都要复制一套的模式。

项目后续应采用三层结构：

### 3.1 通用组网能力

这些能力对大多数游戏复用：

- n2n 虚拟局域网；
- TCP 端口代理；
- UDP 端口代理；
- UDP 广播桥；
- 好友邀请包；
- 诊断报告。

### 3.2 游戏方案

大部分游戏不需要独立页面，只需要一个“游戏方案”描述：

- 游戏名称；
- 识别规则；
- 默认端口；
- 推荐连接方式；
- 是否需要 TCP；
- 是否需要 UDP；
- 是否依赖 LAN 广播；
- 是否支持直接 IP 加入；
- 操作说明。

用户或管理员确认某个游戏的联机类型后，可以沉淀成游戏方案。之后其他用户扫描到同一个游戏，就可以直接复用推荐方案。

### 3.3 少量专用向导

只有当某个游戏确实有复杂流程时，才值得做专用向导。例如：

- 需要选择世界文件；
- 需要启动专用服务端；
- 需要解析服务端输出；
- 需要自动生成配置；
- 需要处理特殊存档、模组或版本。

Terraria 属于这种类型，所以先做了专用向导。Minecraft Java 未来也可能需要类似“服务端包/版本/EULA/server.properties”的半专用流程，但大多数普通 LAN 游戏不应该单独做一个页面。

## 4. 快速开始

### 4.1 房主

1. 打开联机助手。
2. 进入“适配器管理”，同步共享适配器库。
3. 进入“游戏扫描”，扫描本机游戏。
4. 在“推荐方案”中选择游戏。
5. 进入“通用组网中心”，填写：
   - community；
   - secret；
   - supernode；
   - 本机虚拟 IP，例如 `10.10.10.2`。
6. 启动 n2n edge，等待状态变为可用。
7. 如果游戏需要服务端，启动游戏服务端。
8. 复制邀请好友包发给加入者。
9. 如果失败，进入“诊断报告”查看原因。

### 4.2 加入者

1. 打开联机助手。
2. 使用房主提供的邀请包。
3. 填写相同的 community、secret、supernode。
4. 设置不同虚拟 IP，例如 `10.10.10.3`。
5. 启动 n2n edge。
6. 在游戏内连接房主虚拟 IP 和端口。

## 5. n2n / supernode 准备

当前版本采用 n2n 作为主要异地组网方式。

### 5.1 supernode

supernode 通常部署在 VPS 上，用于帮助 edge 节点注册和发现。

示例：

```bash
supernode -p 7777 -vm
```

确认监听：

```bash
ss -lunp | grep 7777
```

### 5.2 edge

edge 运行在玩家电脑上，用于创建虚拟网卡和虚拟 IP。

房主和加入者需要：

- 使用同一个 supernode；
- 使用同一个 community；
- 使用同一个 secret；
- 使用不同虚拟 IP。

示例：

```text
房主：10.10.10.2
加入者：10.10.10.3
```

## 6. Terraria 当前流程

Terraria 是当前首个重点验证游戏。

房主流程：

1. 启动 n2n；
2. 确认房主虚拟 IP 可用；
3. 进入 Terraria 向导；
4. 选择 TerrariaServer.exe；
5. 选择世界；
6. 在程序内启动服务端；
7. 确认 7777 端口监听；
8. 复制邀请包；
9. 好友通过 Join via IP 连接房主虚拟 IP。

当前已完成单机房主侧最小链路验证，但真实双机加入仍建议继续补测。

## 8. 当前能力边界

当前版本不承诺：

- 所有游戏都能一键联机；
- 所有不能本地联机的游戏都能被转换成本地联机；
- 绕过正版验证、反作弊、官方账号系统或平台限制；
- Steam Relay / Steamworks 方案已经可用；
- 未审核游戏方案一定正确；
- UDP 广播桥一定适配所有依赖房间列表的游戏；
- 单机验证等同于真实双机验证。
