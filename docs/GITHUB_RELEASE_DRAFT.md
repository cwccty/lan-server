# GitHub Release 草稿：联机助手 v0.2.0 候选版

> 这是 `v0.2.0` 候选版，重点是让普通用户更容易看懂“现在该做什么”、让维护者更容易复核发布质量。它不是“所有游戏一键可用”的最终版。

## 这个版本是什么

联机助手是面向 Windows 玩家和朋友联机的小工具。它把开房、加入、方案选择、诊断修复和特殊连接工具整理到一个桌面客户端里，尽量减少“我该先开游戏还是先启动组网”的困惑。

## 版本重点

### 已纳入的基础能力

- `v0.1.1` 已加入本地账号、登录/退出、昵称展示、记住我和外观个性化设置。
- 本版继续沿用这些基础能力，并把主要精力放在联机路径说明、方案库信息密度和发布候选验证上。

### v0.2.0 候选新增/强化

- 首页恢复适度信息密度：显示当前状态、房主下一步、加入者下一步和网络拓扑状态。
- 开房/加入说明更明确：先选游戏，再准备组网或服务端，再检测端口/房间，最后复制完整邀请给好友。
- 方案库补齐多游戏说明：Terraria、Palworld、Minecraft Java、Stardew Valley、Cuphead 都有更清楚的适用场景、建议方式和失败后去哪里修。
- 方案库同步预检和诊断建议更清楚：遇到缺少房间名、密钥、邀请信息不完整等情况，会提示重新复制完整邀请、检查组网设置或进入诊断报告。
- 特殊连接工具页补充普通用户说明：端口代理、UDP 广播桥、通用服务端分别说明适用场景、启动前检查和启动后自测。
- 新增 `Steam 中继 / P2P（实验）` 入口：当前用于合法 Steamworks 预检，显示 Steam 客户端、SDK 目录、运行库、AppID、不可用原因和下一步配置说明。

## 快速开始

1. 下载 `LanHelper-v0.2.0-windows-x64.zip`。
2. 解压到一个普通目录，例如桌面或 `D:\Apps\LanHelper`。
3. 运行 `lan-helper.exe`。
4. 首次使用建议先进入“方案库”，同步最新游戏方案。
5. 进入“游戏扫描”或“开房邀请”，选择你要玩的游戏。
6. 如果你是房主：按页面步骤启动组网/服务端，检测端口或房间后，复制完整邀请发给好友。
7. 如果你是加入者：粘贴房主发来的完整邀请，按提示进入同一联机房间，再到游戏里连接页面给出的地址或方式。
8. 遇到失败时，先打开“诊断报告”，优先处理“先修这 1-3 件事”。

## 适合测试的内容

- 首页是否能让你看懂下一步该做什么。
- 开房邀请是否能告诉房主什么时候启动游戏、什么时候创建房间、复制什么给好友。
- 加入流程是否能告诉加入者先粘贴邀请、再启动组网、最后到游戏里填什么。
- 方案库中五款示例游戏的建议是否容易理解。
- 特殊连接工具页是否能解释这些工具什么时候用、失败后如何自测。
- Steam 实验入口是否能诚实显示当前不可用原因，而不是假装已经能联机。

## 已知边界

- Steam 中继 / P2P 当前只是合法预检和后续入口，不代表真实 Steam P2P 联机已经完成。
- 未配置 Steamworks SDK、AppID、Steam 登录和双机/双账号环境时，不能启动真实 Steam P2P 连接。
- Palworld、Minecraft、Stardew Valley、Cuphead 尚未完成真实双机回归，不能声明实机通过。
- 不承诺所有游戏都能一键开服或一键联机；不同游戏仍可能需要官方服务端、游戏内创建房间、远程同屏或手动端口确认。
- 本工具不会修改游戏文件，不绕过 Steam 或游戏拥有权，不复制游戏目录 DLL。

## 建议反馈内容

如果你参与测试，请反馈：

- 游戏名称和版本；
- 你是房主还是加入者；
- 使用的是开房邀请、方案库建议、特殊连接工具还是远程同屏；
- 失败时诊断报告显示的“先修这 1-3 件事”；
- 你最终是在游戏里填写了什么地址或选择了什么加入方式；
- 截图或录屏中请避免包含私人账号、密钥或聊天内容。
