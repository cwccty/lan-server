# v0.2.0 真实双机回归 Runbook

日期：2026-06-08
状态：待执行

## 目的

本 Runbook 用于验证 `v0.2.0` 是否真的满足普通用户关心的三件事：

1. 确实能联机：房主和加入者在两台真实 Windows 机器上进入同一局或同一游玩会话。
2. 流畅联机：进入后能稳定游玩一段时间，并记录延迟、掉线、重连和卡顿情况。
3. 简便联机：用户能按联机助手内的说明完成操作，失败时能靠诊断报告或页面提示继续修复，而不是离开软件到处查资料。

这不是单机 smoke，也不是只验证应用能启动。未执行本 Runbook 前，不得宣传“已双机验证完成”。

## 通用证据要求

每款游戏至少保留以下证据：

- 两台机器的截图或录屏：能看出房主与加入者各自操作。
- 房主/加入者操作时间线：记录每一步开始和完成时间。
- 联机助手诊断报告：失败时必须保存报告，成功时也建议保存一次基线报告。
- 游戏内成功加入截图：能看到加入者已经进入房主的世界、服务器、农场或游玩会话。
- 延迟、掉线、重连记录：至少记录 10 分钟游玩期间是否掉线、是否需要重连、体感延迟。
- ZIP SHA256 和 EXE 版本：确认测试对象确实是本次候选包。

## Palworld 回归步骤

定位：专用服务器或房主侧服务端路线，加入者通过房主给出的联机地址和端口加入。默认端口参考 `8211`，以实际服务器配置为准。

### 房主步骤

1. 确认双方游戏版本一致，房主准备 Palworld Dedicated Server 或等价服务端环境。
2. 启动联机助手，进入方案库并选择 Palworld。
3. 按页面提示启动或确认组网服务，让加入者能进入同一联机房间。
4. 启动 Palworld 服务端，确认端口和服务端状态。
5. 在联机助手中检测房间或端口状态。
6. 复制完整邀请给加入者，邀请中应包含房主联机地址、端口和必要说明。
7. 记录服务端日志、联机助手诊断报告和房主侧截图。

### 加入者步骤

1. 启动联机助手，粘贴房主的完整邀请。
2. 按提示加入同一联机房间。
3. 打开 Palworld，在游戏内输入页面给出的房主联机地址和端口。
4. 进入服务器后截图，并记录进入耗时和游玩稳定性。

### 通过标准

- 加入者能进入房主准备的 Palworld 服务器。
- 双方至少游玩 10 分钟并记录掉线、重连和体感延迟。
- 若失败，诊断报告能指出下一步，例如检查服务端是否启动、端口是否一致、防火墙是否允许。

## Minecraft Java 回归步骤

定位：可测试“开放到局域网”或 Java 服务端路线。默认服务端端口参考 `25565`，开放到局域网时以游戏实际显示端口为准。

### 房主步骤

1. 确认双方 Minecraft Java 版本、模组、资源包和登录状态一致或兼容。
2. 启动联机助手，选择 Minecraft Java。
3. 选择本次路线：游戏内开放到局域网，或启动 Java 服务端。
4. 按联机助手提示准备组网服务，让加入者进入同一联机房间。
5. 如果使用服务端，确认服务端端口；如果使用开放到局域网，记录游戏显示的端口。
6. 复制完整邀请给加入者。

### 加入者步骤

1. 粘贴邀请并进入同一联机房间。
2. 打开 Minecraft Java，选择多人游戏。
3. 输入联机助手给出的房主联机地址和端口。
4. 成功进入世界后截图并记录游玩稳定性。

### 通过标准

- 加入者能进入房主的 Minecraft 世界或服务端。
- 双方至少游玩 10 分钟，记录延迟、掉线和重连情况。
- 失败时能通过诊断报告定位到端口、服务端、防火墙或版本不一致等原因。

## Stardew Valley 回归步骤

定位：合作房间路线。不同版本、平台和联机方式可能使用邀请码、好友邀请或地址加入，必须记录实际入口。

### 房主步骤

1. 确认双方 Stardew Valley 版本、平台和模组状态。
2. 在游戏内创建或加载合作农场。
3. 启动联机助手，选择 Stardew Valley，并按提示准备同一联机房间。
4. 记录游戏内实际提供的加入方式：邀请码、好友邀请、地址加入或其它方式。
5. 将完整邀请和游戏内加入信息发给加入者。

### 加入者步骤

1. 粘贴联机助手邀请并进入同一联机房间。
2. 打开 Stardew Valley，按房主提供的实际方式加入合作农场。
3. 成功进入农场后截图，记录版本差异、进入耗时和稳定性。

### 通过标准

- 加入者能进入房主合作农场。
- 记录实际使用的是邀请码、好友邀请还是地址加入。
- 双方至少游玩 10 分钟，记录延迟、掉线和重连情况。
- 如果版本或平台导致入口不同，必须写入结果，不用一个结论覆盖所有版本。

## Cuphead 回归步骤

定位：这不是输入地址加入的游戏，不应伪装成 LAN 成功。优先验证远程同屏、Steam Remote Play 或类似本地合作远程游玩的路线。

### 房主步骤

1. 确认 Cuphead 版本和控制器/键盘输入方案。
2. 启动联机助手，选择 Cuphead，确认页面提示这是本地合作/远程同屏路线。
3. 启动游戏并开启本地双人准备状态。
4. 启动远程同屏或平台提供的远程一起玩能力。
5. 邀请加入者进入远程游玩会话。

### 加入者步骤

1. 接受远程游玩邀请。
2. 确认输入设备可用，能控制第二名玩家。
3. 进入关卡并记录画面延迟、输入延迟、音频和掉线情况。

### 通过标准

- 加入者能通过远程同屏或平台远程一起玩方式参与游戏。
- 文档和 UI 不把 Cuphead 描述为输入地址加入的游戏。
- 至少完成 10 分钟游玩记录，包含输入延迟和画面稳定性。
