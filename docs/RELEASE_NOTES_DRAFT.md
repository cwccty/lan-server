日期：2026-06-03
版本：0.1.0 草稿

> 本文件是发布前草稿，不等于最终公告。它用于明确当前版本已经验证的能力、仍需用户注意的边界，以及首批游戏的推荐使用方式。

## 1. 这个版本解决什么问题

联机助手的目标不是做聊天房间，也不是替代游戏本身，而是把小型游戏联机中经常分散在不同工具里的步骤整理到一个客户端中：

```text
游戏扫描
→ 适配器匹配
→ 推荐联机方案
→ n2n 组网
→ TCP / UDP / UDP 广播桥辅助
→ 游戏服务端 / 邀请包
→ 诊断报告
```

当前版本重点验证的是：

- n2n edge 与 VPS supernode 的异地组网；
- Terraria 专用服务端托管与 Join via IP；
- TCP 端口代理；
- UDP 单播端口代理；
- UDP 广播桥；
- 适配器共享库同步；
- 当前游戏上下文诊断；
- 本地 adapter 草稿与共享库沉淀流程。

## 5. 用户需要准备什么

### 房主需要

- Windows 客户端；
- n2n edge 已随工具放置或可检测；
- 一个可用 VPS supernode；
- 游戏本体；
- 如果是 Terraria，需要 TerrariaServer.exe 和世界文件。

### 加入者需要

- 同版本客户端；
- 与房主相同的 community；
- 与房主相同的 secret；
- 同一个 supernode；
- 不同的虚拟 IP，例如房主 `10.10.10.2`、加入者 `10.10.10.3`。

## 6. 快速使用流程

### 6.1 房主

1. 打开联机助手。
2. 进入“适配器管理”，同步共享适配器库。
3. 进入“游戏扫描”，选择游戏。
4. 进入“推荐方案”，查看该游戏需要什么能力。
5. 进入“通用组网中心”，填写 community、secret、supernode、本机虚拟 IP。
6. 启动 n2n edge，等待 ACK / PONG / [OK]。
7. 如果是 Terraria，进入 Terraria 向导，在程序内启动服务端。
8. 复制邀请好友包发给加入者。
9. 如遇问题，进入诊断报告生成报告。

### 6.2 加入者

1. 打开联机助手。
2. 使用房主邀请包中的 community、secret、supernode。
3. 设置不同虚拟 IP。
4. 启动 n2n edge，等待 ACK / PONG / [OK]。
5. 在游戏内连接房主虚拟 IP 和端口。

## 7. 常见失败原因

### 7.1 supernode 不响应

可能原因：

- VPS 没有启动 supernode；
- 防火墙没有放行端口；
- 客户端填写的 IP / 端口错误；
- 网络无法访问 VPS。

建议：

- 在 VPS 上执行 `ss -lunp | grep 7777`；
- 确认使用 `supernode -p 7777 -vm` 或 systemd 服务；
- 客户端重新启动 n2n edge 后等待 10 到 20 秒。

### 7.2 authentication error / IP or MAC already in use

可能原因：

- 两台电脑使用了相同虚拟 IP；
- 上一次异常退出后 supernode 未释放旧注册；
- community / secret 不一致。

建议：

- 房主使用 `10.10.10.2`；
- 加入者使用 `10.10.10.3`；
- 双方 community / secret 完全一致；
- 停止 edge 后等待一会再启动。

### 7.3 游戏端口不可达

可能原因：

- 游戏服务端未启动；
- 服务端只监听 `127.0.0.1`；
- 防火墙拦截；
- 需要 TCP 端口代理。

建议：

- 房主先测试 `127.0.0.1:游戏端口`；
- 再测试 `房主虚拟 IP:游戏端口`；
- 如仅本机可达，尝试 TCP 端口代理：`0.0.0.0:端口 -> 127.0.0.1:端口`。

### 7.4 房间列表看不到

可能原因：

- 游戏依赖 LAN 广播/组播发现；
- 虚拟局域网只解决 IP 可达，不保证广播发现；
- 需要 UDP 广播桥。

建议：

- 如果游戏支持直接 IP 加入，优先使用房主虚拟 IP；
- 如果只能依赖房间列表，尝试 UDP 广播桥；
- 广播桥只能辅助发现，不保证最终加入成功。

## 8. 当前能力边界

当前版本可以做：

- n2n 组网；
- VPS supernode 连接；
- TCP 端口代理；
- UDP 单播代理；
- UDP 广播桥；
- Terraria 服务端托管；
- 适配器共享库同步；
- 当前游戏上下文诊断；
- 邀请好友包生成。

当前版本不承诺：

- 所有游戏都能一键转 LAN；
- 绕过正版验证、反作弊或官方账号服务；
- Steam Relay 插件已经可用；
- 未经审核的游戏 adapter 一定正确；
- 单机验证等同于双机真实联机；
- UDP 广播桥一定适配所有依赖房间列表发现的游戏。

## 9. 发布结论草案

当前版本可以作为“首个公开测试版 / MVP 验证版”发布，但建议发布文案使用以下定位：

```text
联机助手 0.1.0 是面向小型游戏联机的 MVP 测试版。
当前已完成 n2n 组网、端口代理、UDP 广播桥、Terraria 房主侧链路、适配器共享库和诊断报告。
真实双机加入、更多游戏 adapter 审核仍在继续。
```

不建议写成：

```text
支持所有局域网游戏一键联机。
所有不能本地联机的游戏都能转换。
无需配置即可异地联机。
```
