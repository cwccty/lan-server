# 两台电脑组网卡住排障 SOP

适用场景：

- A 电脑启动组网后一直显示“组网程序已启动，但中继尚未确认”。
- B 电脑显示“已配置未启动 / 组网服务未运行”。
- 双方已关闭防火墙但仍无法继续。

## 先收集证据

两台电脑都进入“加入与组网”或“诊断报告”，分别复制：

1. 完整诊断报告；
2. 手动启动命令；
3. 组网日志。

不要只截图“启动中”，必须同时拿到中继地址、房间名、联机地址、PID、最后错误和最近日志。

## 对照顺序

1. **中继地址**
   - 两台电脑必须完全一致，包括端口。
   - 一台为空或端口不同，都会导致无法确认。

2. **房间名和密钥**
   - 两台电脑必须使用同一份邀请包或同一组房间凭证。
   - 注意多余空格、全角字符、输入法自动替换。

3. **联机地址**
   - 两台电脑的 10.x 联机地址必须不同。
   - 如果相同，给其中一台换一个地址后停止组网并重新启动。

4. **已配置未启动**
   - 重点看：
     - 组网程序文件是否找到；
     - 记录 PID 是否存在；
     - PID 是否存活；
     - 最后错误；
     - 组网日志。
   - 常见原因：组网程序缺失、启动后立即退出、权限不足、虚拟网卡异常。

5. **中继尚未确认**
   - 重点看：
     - 中继地址是否可达；
     - 房间名/密钥是否一致；
     - 组网日志是否有拒绝、超时、认证失败、地址冲突。
   - 这不是“继续等一会”能解决的状态。

6. **网卡/TAP/Wintun 错误**
   - 优先修复组联网卡。
   - 必要时用管理员身份运行联机助手。

## 结论判断

- 双方报告里中继地址、房间名、密钥一致，且联机地址不同，但仍没有中继确认：优先排查中继地址/端口可达性。
- 一方有 PID、一方没有 PID：没有 PID 的机器先修复启动失败。
- 日志出现认证/密钥错误：重新生成邀请包，不要手填。
- 日志出现 IP/MAC 冲突：更换其中一台联机地址。
- 日志出现 TAP/Wintun/网卡错误：先修复虚拟网卡。
