## 复测目标

验证真实用户反馈的两类状态是否已经可诊断、可分流、可回传证据：

1. 一台电脑启动组网后长期卡住。
2. 另一台电脑显示“已配置未启动”。
3. 注册修复无效时，用户仍能按 UI 提示继续排查。

## 验收判断

- 若显示“已配置未启动”：优先判定为本机组网程序、权限、PID、虚拟网卡或启动命令问题。
- 若显示“中继尚未确认”：优先判定为中继地址、房间名、密钥、联机地址冲突或 UDP/网络环境问题。
- 若两边参数一致但仍中继未确认：要求至少一方换手机热点复测，以区分中继问题和本地网络拦截。

## 不能接受的回传

- 只说“还是不行”，没有诊断报告。
- 只截图“启动中”，没有双方中继地址、房间名、联机地址和日志。
- 只说“防火墙关了”，没有手动启动命令和组网日志。
