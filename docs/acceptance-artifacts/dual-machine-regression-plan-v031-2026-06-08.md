# v0.3.1 真实双机/实机回归计划（未完成项记录）

日期：2026-06-08
结论：本文件是回归计划与证据清单，不代表 Palworld、Minecraft Java、Stardew Valley、Cuphead 已实机通过。

## 通用记录要求

每个游戏至少记录：
- 房主机器：Windows 版本、联机助手版本、游戏版本、网络环境、是否管理员运行。
- 加入者机器：Windows 版本、联机助手版本、游戏版本、网络环境。
- 使用路线：组网直连、Steam/P2P、服务端开服、端口代理、UDP 广播桥或远程同屏。
- 关键输入：房主地址、端口、服务端文件路径、Steam 连接工具文件夹、邀请文本。
- 成功证据：双方截图、联机助手诊断日志、游戏内玩家列表或加入成功画面。
- 失败证据：错误提示、端口检测、进程状态、双方诊断报告。

## Palworld / 幻兽帕鲁

状态：待实机验证。
建议路线：服务端开服；默认 UDP 8211；必要时结合 Steam/P2P 或端口/防火墙诊断。

房主步骤：
1. 准备 Palworld Dedicated Server 或官方服务端文件。
2. 在一键开服向导选择 Palworld，填写服务端可执行文件路径。
3. 运行“检查文件”，确认路径、工作目录、端口占用检查通过。
4. 启动服务端，观察进程状态与 UDP 8211 端口线索。
5. 复制邀请与诊断文本发给加入者。

加入者步骤：
1. 确认游戏版本与房主一致。
2. 按邀请中的地址/端口在游戏内加入。
3. 加入失败时先回传游戏错误、联机助手诊断文本和网络路线。

## Minecraft Java

状态：待实机验证。
建议路线：服务端开服；默认 TCP 25565；需要 Java 和 `eula.txt=true`。

房主步骤：
1. 准备 `server.jar` 或可启动的 `.bat/.cmd`。
2. 确认 Java 在 PATH 中可用，并处理 `eula.txt`。
3. 在一键开服向导选择 Minecraft Java，运行“检查文件”。
4. 启动服务端，确认 TCP 25565 监听或配置的端口可检测。
5. 复制 `IP:端口` 邀请和诊断文本。

加入者步骤：
1. Minecraft Java 版本与服务端一致。
2. 使用多人游戏直接连接 `IP:端口`。
3. 失败时优先检查服务端日志、端口监听、防火墙、组网 IP。

## Stardew Valley

状态：待实机验证。
建议路线：优先游戏自带联机/虚拟局域网直连；失败后再按诊断建议尝试桥接。

房主步骤：
1. 开启游戏合作模式并记录房主地址/邀请方式。
2. 确认双方处于同一虚拟组网或可互通网络。
3. 复制联机助手诊断文本。

加入者步骤：
1. 在游戏内使用合作/加入入口。
2. 如果看不到房间列表，尝试 UDP 广播桥二选一路线并记录结果。

## Cuphead

状态：待实机验证。
建议路线：远程同屏优先；不要伪装成局域网直连已支持。

房主步骤：
1. 准备 Steam Remote Play 或 Sunshine + Moonlight 等远程同屏方案。
2. 联机助手仅提供路线说明与诊断入口，不宣称改造为局域网联机。

加入者步骤：
1. 通过远程同屏邀请加入。
2. 记录延迟、手柄输入、画面和连接稳定性。
