# Stitch 提示词：联机助手 iOS 高级清爽交互式前端

> 使用方式：把本文件完整复制给 Stitch。目标是重新生成一套完整前端设计，不是改色，不是单页，不是只做首页。必须一次性覆盖所有页面、所有关键按钮、所有功能入口和主要状态。

## 任务目标

请为 Windows 桌面端产品“联机助手”重新设计一套完整前端。要求类似 iOS / macOS 原生应用的高级感、清爽感、轻量玻璃材质、优秀留白、明确层级、强交互感。

不要沿用深蓝科技工具风。不要游戏加速器式霓虹风。不要营销海报风。不要只换颜色。请重新组织界面、组件、流程、按钮和状态。

## 产品定位

联机助手是面向普通 PC / Steam 玩家、小型游戏房主、加入者的 Windows 小型游戏联机辅助工具。

它帮助用户完成：

- 更新共享游戏方案库；
- 扫描本机游戏；
- 根据游戏推荐联机方案；
- 配置并启动 n2n 组网；
- 配置 TCP 端口代理；
- 配置 UDP 端口代理；
- 配置 UDP 广播桥；
- 使用 Terraria 向导启动服务端；
- 分配好友虚拟 IP；
- 生成邀请好友包；
- 生成诊断报告。

核心原则：

1. 告诉用户下一步做什么；
2. 绿色成功状态必须来自真实检测；
3. 高级功能要渐进展示，不要一屏压满配置；
4. 失败状态必须给出原因和下一步建议；
5. 普通用户不应该看到太多 adapter、registry、MVP、PoC 这类内部词。

## 视觉方向

请使用类似 iOS / macOS 高级工具应用的设计语言：

- 背景：低饱和暖灰、雾面白、浅石墨、半透明磨砂层；
- 主色：香槟金 / 柔和琥珀 / 淡银灰；
- 辅助色：少量青绿色表示成功，少量红色表示错误，少量蓝色只用于链接或技术信息；
- 卡片：轻量半透明、细边框、柔和阴影，像 macOS 设置面板 / iOS 控制中心；
- 字体：中文优先，接近 SF Pro / PingFang SC / Microsoft YaHei UI；
- 圆角：按钮 12px，卡片 18-24px，大容器 28px；
- 布局：清爽留白，多用分组、列表、步骤条、分段控制、底部操作条；
- 动效：轻微 hover、按压、切换、展开收起、进度环；
- 不要大量深蓝色；
- 不要密集大段技术说明；
- 不要重复的普通卡片网格；
- 不要所有模块都长得一样。

## 全局 App Shell

请设计一套统一壳层：

### 左侧导航栏

类似 macOS 应用侧栏：

- 顶部品牌：联机助手；
- 副标题：小型游戏联机工具；
- 导航分组：
  - 开始：首页；
  - 准备：方案库、游戏扫描；
  - 执行：推荐方案、通用组网中心、Terraria 向导；
  - 排查：诊断报告；
  - 系统：设置 / 帮助。
- 当前页面高亮使用柔和玻璃选中态；
- 底部显示当前建议路径：方案库 → 扫描游戏 → 推荐方案 → 通用组网。

### 顶部指挥栏

每页主内容顶部都有：

- 当前页面名；
- 一句话说明；
- 快捷按钮：打开诊断、启动组网；
- 当前状态 badge，例如：待配置 / 运行中 / 可邀请 / 需要诊断。

### 通用组件

必须设计以下组件并在页面中复用：

- 分段控制：我是房主 / 我是加入者；
- 4 步流程条；
- 状态卡：通过 / 警告 / 失败 / 未检测；
- 进度环 / 准备度环；
- 任务清单；
- 主要配置卡；
- 高级功能抽屉；
- 邀请包预览；
- 诊断报告卡；
- 空状态；
- 加载遮罩；
- 同步结果表格；
- 终端日志预览；
- 复制成功 toast。

## 必须一次性设计所有页面

请一次性生成所有页面，不要只生成首页，不要让我逐页追加。每个页面都要包含关键按钮和功能状态。

# 1. 首页

目标：第一次打开就知道该怎么开始。

必须包含：

- 主标题：从“能不能连”到“下一步怎么做”；
- 角色切换：我是房主 / 我是加入者；
- 场景选择卡：
  - 直接 IP 联机；
  - 专用服务端；
  - 房间列表发现；
- 准备度圆环：房主准备度 / 加入者准备度；
- 动态步骤列表：
  - 房主：更新方案库、扫描游戏、启动 n2n、启动服务端/房间、复制邀请包；
  - 加入者：接收邀请、填写相同组网信息、使用不同虚拟 IP、启动 n2n、连接房主虚拟 IP；
- 联机拓扑图：房主、supernode、好友；
- 准备清单：方案库、游戏、组网、端口；
- 主要按钮：
  - 开始准备房间；
  - 填写好友邀请；
  - 连接失败诊断；
  - 扫描本机游戏。

交互要求：

- 切换身份后，准备度和步骤列表变化；
- 切换联机场景后，右侧说明和推荐按钮变化；
- 鼠标 hover 卡片有轻微浮起；
- 点击准备清单项应跳转到对应页面。

# 2. 方案库页面

目标：普通用户一键更新共享方案，高级用户可管理本地 adapter。

必须包含：

- 页面标题：游戏方案库；
- 主按钮：一键更新共享方案；
- 次按钮：同步本地示例库、恢复默认共享库地址；
- 状态卡：
  - 共享库状态；
  - 上次同步时间；
  - 新增数量；
  - 更新数量；
  - 跳过数量；
- 同步结果表格：游戏名、状态、原因、来源；
- 共享库地址输入框；
- 高级区域：导入 JSON、导出 JSON、创建/编辑方案；
- 空状态：尚未同步方案，提示点击一键更新共享方案。

按钮要覆盖：

- 一键更新共享方案；
- 同步本地示例库；
- 恢复默认共享库地址；
- 同步指定共享库；
- 保存方案；
- 导入方案；
- 导出方案。

---

# 3. 游戏扫描页面

目标：扫描本机游戏并选择本次要联机的游戏。

必须包含：

- 页面标题：游戏扫描；
- 主按钮：扫描本机游戏；
- 次按钮：刷新 Steam 游戏、打开方案库；
- 加载状态：正在扫描 Steam 与本地路径；
- 游戏列表卡片：
  - 游戏名；
  - 安装路径；
  - 方案来源；
  - 默认端口；
  - 联机能力；
  - 风险/置信度；
- 每个游戏卡按钮：
  - 查看推荐方案；
  - 查看详情；
  - 创建方案草稿；
- 空状态：未扫描到游戏，可手动进入通用组网或创建方案草稿。

交互要求：

- 支持搜索框；
- 支持筛选：全部 / 已有方案 / 待确认 / 可直连 / 需要服务端；
- hover 游戏卡时显示下一步按钮。

# 4. 推荐方案页面

目标：告诉用户当前游戏下一步怎么做。

必须包含：

- 页面标题：推荐方案；
- 4 步流程条：先组网、开游戏服务、发邀请、失败诊断；
- 开始前确认：双方虚拟 IP 不重复、房主先启动服务端、优先直连房主虚拟 IP；
- 执行清单：
  - 适配器判断；
  - 通用组网；
  - 游戏启动/服务端；
  - 本机端口监听；
  - 邀请好友；
- 游戏邀请好友包区域：
  - 好友昵称输入框；
  - 当前建议 IP；
  - 分配 / 选择好友虚拟 IP；
  - 检测好友连接；
  - 好友分配列表；
  - 是否包含 n2n 密钥开关；
  - 邀请包预览；
  - 复制游戏邀请好友包；
- 游戏摘要：游戏、路径、方案来源、默认端口；
- 下一步卡：带入参数并进入通用组网。

按钮必须包含：

- 刷新执行清单；
- 分配 / 选择好友虚拟 IP；
- 检测好友连接；
- 复制游戏邀请好友包；
- 先刷新检测状态；
- 带入参数并进入通用组网；
- 执行推荐启动项。

  - 本机虚拟 IP；
  - 对方 / 房主虚拟 IP；
  - 游戏端口；
- 按钮：
  - 保存 n2n 配置；
  - 启动 n2n edge；
  - 停止 n2n edge；
  - 刷新组网状态；
  - 复制给朋友的通用组网配置；

- 当前 n2n 状态区域；

# 6. Terraria 向导页面

目标：帮助 Terraria 房主在程序内启动服务端，并让加入者知道怎么连。

必须包含：

- 页面标题：Terraria 向导；
- 角色切换：我是房主 / 我是加入者；
- 4 步流程条：配置 n2n、选择世界、启动服务端、复制邀请；

- 服务端配置：
  - 世界编号；
  - 世界文件路径；
  - 最大人数；
  - 游戏端口；
  - 服务端密码；
  - 自动端口转发；
- 按钮：
  - 保存配置并启动 n2n edge；
  - 在程序内启动服务端；
  - 停止内嵌服务端；
  - 一键自检；
  - 复制邀请信息；
  - 复制自检结果；
- 服务端状态卡：运行中/未运行、PID、端口监听、最近日志；
- 内嵌服务端控制台；
- 加入者说明：使用房主虚拟 IP Join via IP。

必须包含：

- 页面标题：诊断报告；
- 按钮：
  - 生成全局诊断；
  - 生成当前游戏诊断；
  - 复制诊断报告；
  - 清空/重新生成；
- 保留上次诊断结果提示；
- 总结卡：
  - 最可能原因；
  - 严重程度；
  - 下一步建议；
- 检查项列表：
  - n2n edge；
  - supernode；
  - 虚拟 IP；
  - TCP 代理；
  - UDP 代理；
  - UDP 广播桥；
  - 当前游戏端口；

- 每项要有状态、证据、建议；
- 诊断时间线；
- 诊断详情 JSON / 文本预览。

交互要求：

- 点击检查项可以展开证据和建议；

- 复制报告有成功 toast。
