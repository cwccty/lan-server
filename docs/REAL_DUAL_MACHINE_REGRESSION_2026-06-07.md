## 通用记录要求

每个游戏至少记录：

1. 房主电脑系统版本、游戏版本、联机助手包 hash。
2. 好友电脑系统版本、游戏版本、是否同一联机房间。
3. 房主端截图：选游戏、启动组网、开房邀请/特殊工具状态、游戏内房间或服务端状态。
4. 好友端截图：加入入口、输入地址/接受邀请、成功进入游戏后的同屏或多人状态。
5. 失败时导出诊断报告，并记录防火墙提示、端口、游戏错误文本。

## Palworld / 幻兽帕鲁

- 主路线：专用服务器 + 虚拟联机房间直连。
- 协议/端口：默认 UDP `8211`，如服务端改过端口，以实际端口为准。
- 不应使用：TCP-only 端口检测作为成功/失败最终判据；UDP 远端连通性最终以游戏内加入为准。

### 房主步骤

1. 两台电脑进入同一联机助手组网房间，确认房主联机地址可见。
2. 房主启动 `Palworld Dedicated Server / PalServer.exe`。
3. 联机助手选择 Palworld，确认端口为 UDP `8211` 或实际服务端端口。
4. 生成邀请包并发给好友。

### 好友步骤

1. 加入同一联机助手房间。
2. 在 Palworld 多人游戏/专用服务器地址里输入 `房主联机地址:8211` 或实际端口。
3. 成功进入服务器并移动 1 分钟，确认无频繁掉线。

### 通过标准

- 好友能进入房主 Palworld 服务端。
- 房主和好友能互相看到角色/世界状态。
- 1 分钟内无断线；失败时提示能指向 UDP、防火墙、服务端未启动或端口不一致。

## Minecraft Java Edition

- 主路线：开放到局域网 / 直接 IP 加入。
- 协议/端口：TCP；“开放到局域网”时必须以游戏聊天框显示端口为准，不默认强制 `25565`。
- 不应要求：普通玩家必须启动 dedicated server。

### 房主步骤

1. 两台电脑进入同一联机助手组网房间。
2. 房主进入 Minecraft Java 世界，按 Esc 选择“开放到局域网”。
3. 记录游戏显示的 LAN 端口，并填入联机助手开房邀请页。
4. 复制邀请包给好友。

### 好友步骤

1. 加入同一联机助手房间。
2. 打开 Minecraft 多人游戏，使用“直接连接”输入 `房主联机地址:游戏显示端口`。
3. 成功进入世界后移动/放置方块 1 分钟。

### 通过标准

- 好友能进入房主世界。
- 端口使用游戏显示端口而不是固定写死。
- 如果失败，诊断能提示检查 Java/Minecraft 防火墙、是否真的开放到局域网、双方是否同房间。

## Stardew Valley

- 主路线：游戏内原生合作优先；仅在当前版本/平台支持 LAN/IP 时使用房主联机地址和游戏端口。
- 协议/端口：方案库默认端口 `24642` 仅作为参考，实际以游戏显示或平台机制为准。
- 不应强行转换：如果游戏只显示平台邀请，不应伪造 LAN 邀请。

### 房主步骤

1. 两台电脑进入同一联机助手组网房间。
2. 房主创建/加载合作农场并保持在线。
3. 如果游戏提供 LAN/IP 或显示端口，把端口填入联机助手；否则使用游戏原生邀请。
4. 将联机助手整理出的说明或游戏原生邀请发给好友。

### 好友步骤

1. 优先使用游戏内合作入口加入。
2. 如果版本支持 IP 加入，输入 `房主联机地址:端口`。
3. 成功进入农场后切换场景/移动 1 分钟。

### 通过标准

- 好友能进入合作农场。
- 软件不会误导用户“必须端口代理/必须 LAN”。
- 失败时提示区分“游戏平台不支持 IP 加入”和“组网/防火墙/端口问题”。

## Cuphead / 茶杯头

- 主路线：本地双人 + Steam Remote Play Together 或 Sunshine + Moonlight。
- 协议/端口：不生成 LAN 端口邀请，不做端口检测作为成功标准。
- 不应使用：虚拟 LAN/IP 加入、服务端端口检测。

### 房主步骤

1. 启动 Cuphead，进入本地双人/同屏合作模式。
2. Steam 版使用 Remote Play Together 邀请好友；非 Steam 或体验不佳时使用 Sunshine 串流。
3. 在联机助手中选择 Cuphead，确认说明走远程同屏路线。

### 好友步骤

1. 接受 Steam Remote Play 邀请，或用 Moonlight 连接房主 Sunshine。
2. 确认手柄/键鼠输入被授权给第二位玩家。
3. 进入关卡并共同操作 1 分钟。

### 通过标准

- 好友可控制第二位玩家。
- 画面/输入延迟可接受；如延迟高，说明引导用户降低分辨率/码率或改用 Sunshine + Moonlight。
- 软件不提示 LAN 端口检测失败。

## 当前受限项

真实双机实机回归需要以下资源，当前本机环境无法代替：

- 第二台 Windows 机器或一名远端好友；
- Palworld、Minecraft Java、Stardew Valley、Cuphead 的可运行客户端/账号；
- Palworld Dedicated Server / Minecraft / Stardew / Cuphead 的实际游戏内操作权限；
- 两端可加入同一联机助手组网房间的网络环境。

在这些资源到位前，只能完成本机前置检查、方案库逻辑检查、release UI 冒烟和打包分发验证；不能声称“真实双机游戏内已通过”。
